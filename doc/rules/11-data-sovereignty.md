# 11. 데이터 주권 및 개인정보 처리 (Data Sovereignty) [MUST]

> 원본: [RULE.md](../RULE.md) 11장. GDPR, CCPA 및 국내 개인정보보호법 2026년 개정안 준수를 위한 기술 요구사항.

### 11.1 데이터 최소화 및 파기 (Data Minimization & Shredding)

- **목적 기반 수집**: 서비스 제공에 필요한 **최소한의 데이터만** 수집. 수집 목적 달성 후 즉시 파기.
- **물리적 파기**: DB 레코드 삭제 시 논리 삭제가 아닌 **물리 삭제(Hard Delete)** 원칙. 법령 보존 필요 시 별도 Cold Storage 이관·통제.

### 11.2 사용자의 권리 보장 (Right to be Forgotten)

- **탈퇴 즉시 처리**: 사용자 탈퇴 시 PII **즉시 익명화 또는 삭제**.
- **연관 데이터 삭제**: 주 저장소뿐 아니라 **로그(ELK), 캐시(Redis), 백업** 내 개인정보 식별·삭제 또는 비가독 변조.

### 11.3 데이터 거버넌스 및 감사 (Audit Trail)

- **DB 직접 접근 통제**: 운영자 DB 직접 조회(DML/DQL) 원칙 금지. 긴급 시 사유·결재 후 일회성 계정만 발급.
- **접근 로그 보존**: 개인정보 접근 기록 **최소 2년 이상** 변조 불가 형태(WORM 등) 보존.

### 11.4 개인정보 마스킹 및 암호화

- **화면 마스킹**: 관리자 UI에서 이메일, 전화번호, 이름 일부 **반드시 마스킹**.
- **검색 가능 암호화**: 검색 필요 시 **결정적 암호화(Deterministic Encryption)** 또는 검색용 해시 컬럼으로 평문 저장 방지.
